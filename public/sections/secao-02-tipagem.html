● Nível Simples Os 5 tipos fundamentais

TypeScript adiciona tipos ao JavaScript. Declarar o tipo de uma variável faz o editor avisar quando você comete erros antes mesmo de executar o código.

  • string— textos: nomes, frases. Ex: "Olá"
  • number— números inteiros ou decimais. Ex: 42, 3.14
  • boolean— verdadeiro ou falso: true / false
  • void— funções que não retornam nenhum valor
  • any— desabilita verificação. Evite — perde o benefício do TypeScript
nivel-simples.ts 
const nomePessoa: string   = "Alice"
const idadePessoa: number   = 30
const estaLogado: boolean   = true

function exibirSaudacao(nome: string): void {
  console.log(`Olá, ${nome}!`)   // não retorna nada → void
}

// ❌ Erro detectado antes de rodar:
// const altura: number = "1.75"  → Type 'string' não é 'number'
● Nível Médio Arrays tipados e tuplas

Em TypeScript, arrays têm um tipo definido para seus elementos. Uma tupla é um array de tamanho fixo onde cada posição tem um tipo específico.

nivel-medio.ts 
// Array tipado
const listaDeNotas: number[] = [8.5, 7.0, 9.2]
const listaDeNomes: string[] = ["Ana", "Bruno", "Carla"]

// Tupla: tamanho e tipos fixos por posição
const coordenadas: [number, number] = [-23.5, -46.6]
const entradaDoBanco: [string, number, boolean] = ["Alice", 5000, true]

// Tipo object
const perfilUsuario: { nome: string; idade: number } = {
  nome: "Carlos",
  idade: 28
}

// null e undefined
let emailOpcional: string | null = null
emailOpcional = "carlos@email.com"   // ✅ ok
● Nível Avançado Tipos genéricos e utility types

Tipos genéricos permitem criar funções e estruturas que funcionam com qualquer tipo. Os utility types são tipos prontos do TypeScript para transformar outros tipos.

nivel-avancado.ts 
// Genérico: função que aceita qualquer tipo
function primeiroElemento<T>(lista: T[]): T {
  return lista[0]
}
primeiroElemento([1, 2, 3])        // retorna: number
primeiroElemento(["a", "b"])       // retorna: string

// Utility Types — transformam tipos existentes
interface ConfiguracaoEmail {
  destinatario: string
  assunto: string
  corpoMensagem: string
}

type ConfiguracaoRascunho = Partial<ConfiguracaoEmail>
// Todos os campos viram opcionais

type ConfiguracaoFinal = Required<ConfiguracaoRascunho>
// Todos os campos voltam a ser obrigatórios

type SomenteDestinatario = Pick<ConfiguracaoEmail, "destinatario">
// Pega só o campo "destinatario"

type SemCorpo = Omit<ConfiguracaoEmail, "corpoMensagem">
// Remove o campo "corpoMensagem"
